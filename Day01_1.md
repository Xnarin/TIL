### 1.3 TIL

<aside>
💻

로컬 저장소 (TIL)

Working Directory // 파일을 작성하고 수정하는 공간

→ Staging area // 변경사항을 준비하는 공간

→ Commits // 확장된 버전이 저장되는 공간

</aside>

Working Directory는 실제로 파일을 만들고 수정하는 프로젝트 폴더이다

Staging Area는 다음 커밋에 포함한 변경 사항을 선택해 두는 공간이다 git add 를 실행하면 변경 사항이 이곳에 등록된다.

Commit은 git commit으로 확정한 버전 기록이 저장되는 공간이다.

→ add 후에도 더 수정하고 싶을 수 있어서

Commit에는 고유한 해시 값이 부여된다.

### 1.4 명령어 정리

<aside>
💻

git init // 현재 폴더를 git 저장소로 만든다

git status

git add .

git commit -m "커밋 메시지" → 로컬 저장소에 버전 기록

git log --oneline

git config --global user.name

git config --global user.email

git remote add origin [원격저장소URL]

git remote -v // 원격 저장소 확인

git push origin master → 로컬 커밋을 원격 저장소로 이동

</aside>

### 1.5  질문 정리

<aside>
💻

 git push origin master 와  git push -u origin master 의 차이는?

→ -u 는 업스트림을 설정하는 옵션, 이를 설정하면 한번 하고나서는 그 다음부터는 git push만 하여도 원격 저장소에 올라간다

</aside>

```mathematica
일반 폴더    │    │ git init    ▼로컬 Git 저장소Working Directory    │    │ git add    ▼Staging Area    │    │ git commit    ▼Commits    │    │ git push    ▼GitHub 원격 저장소
```

## 2. CLI

### 2.1 GUI (Graphic User Interface)

그래픽을 통해 사용자와 컴퓨터가 상호작용

### 2.2 CLI (Command Line Interface)

터미널을 통해 사용자와 컴퓨터가 상호작용

### 2.3 Why CLI?

CLI는 성능을 더 적게 소모하며, 세부적인 기능을 사용 가능

수 많은 서버나 개발 시스템이 CLI환경이다
